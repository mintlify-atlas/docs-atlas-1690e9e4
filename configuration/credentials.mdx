title: "Configure Instagram credentials for Osintgram login"
description: "Set up your Instagram username and password in credentials.ini, or use the make setup command to populate the file interactively from the terminal."
---

Osintgram reads your Instagram login details from a single INI file before it attempts any API calls. You must populate this file before running the tool — Osintgram will exit immediately with an error if the fields are blank or missing.

## The credentials.ini file

The file lives at `config/credentials.ini` inside the repository root. Its structure is:

```ini config/credentials.ini
[Credentials]
username =
password =
hikerapi_token =
```

| Field | Required | Description |
| --- | --- | --- |
| `username` | Yes (without HikerAPI) | Your Instagram username (handle without `@`) |
| `password` | Yes (without HikerAPI) | Your Instagram account password |
| `hikerapi_token` | No | HikerAPI token — if set, Osintgram uses HikerAPI instead of the private API |

## Populate credentials

<Tabs>
  <Tab title="Manually">
    Open `config/credentials.ini` in any text editor and fill in your details:

    ```ini config/credentials.ini
    [Credentials]
    username = your_instagram_handle
    password = your_password
    hikerapi_token =
    ```
  </Tab>
  <Tab title="make setup">
    Run the setup target from the repository root. It prompts you for each value and writes the file for you:

    ```bash
    make setup
    ```

    This is the recommended method when using Docker or if you want to avoid touching the file directly.
  </Tab>
</Tabs>

<Warning>
  Never commit `config/credentials.ini` to a public repository. The file contains your plaintext Instagram password. Add it to `.gitignore` if you fork the project.
</Warning>

<Warning>
  Do not use your primary personal Instagram account. Instagram may flag or temporarily restrict an account that makes automated API requests. Use a secondary or burner account instead.
</Warning>

## Session cache

After a successful login, Osintgram saves a session token to `config/settings.json`. On subsequent runs the tool reuses this cached session rather than re-authenticating, which reduces the chance of triggering Instagram's security checks.

```
config/
├── credentials.ini   # Your username and password
└── settings.json     # Created automatically on first login
```

You do not need to create or edit `config/settings.json` yourself. Osintgram manages it automatically via `onlogin_callback` in `src/Osintgram.py`.

### Clear the session cache

If your session becomes stale or you switch accounts, pass the `--cookies` / `-C` flag to force a fresh login:

```bash
python3 main.py <target_username> --cookies
# or
python3 main.py <target_username> -C
```

This calls `clear_cache()` before logging in, deleting the existing `config/settings.json` so a new session is negotiated.

## Troubleshooting

### challenge_required error

Instagram raises a `challenge_required` error when it detects behavior it considers suspicious — for example, logging in from an unfamiliar location or making an unusually large number of requests. When this happens, Osintgram prints a URL:

```
Please follow this link to complete the challenge: https://www.instagram.com/challenge/...
```

Open that URL in a browser while signed in as the account specified in `credentials.ini`, then complete the verification step Instagram presents (entering a code sent by email or SMS, confirming your identity, etc.). Once the challenge is cleared, re-run Osintgram normally.

<Tip>
  Using a burner account reduces the frequency of challenge prompts because its activity pattern is less likely to conflict with your day-to-day personal Instagram usage.
</Tip>
